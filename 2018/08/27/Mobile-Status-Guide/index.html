Mobile Status Guide

Posted on 2018-08-27 | In work |

원리

  1. PDA의 네트워크 연결시 PDA는 DHCP에 hostName을 전달
  2. DHCP는 해당 PDA에 Dynamic IP address 부여.
  3. 네트워크 접속 성공후 PDA가 veles2pda에 접속시 IP 정보만 전달.
  4. Veles2pda은 이런 IP를 가진 Device가 접속되었다고 aJax를 통해 asset web app의 REST API를 POST Call 한다.
  5. Asset web app은 이 veles2pda에서 전달한 IP 정보로 hostName을 네트워크에 문의하여 찾아오고 가지고 있는 HostName리스트를 참조하여 로그를 기록한다.
  6. DHCP lease 기간동안은 dl Dynamic IP 와 hostName이 Key vs Value처럼 look up이 가능하다.

추가 업데이트 계획

  • 각Location별로 변경이 필요한 부분은 고쳐서 쓰세요
  • 관련 파일 : (\HKE.LOCAL\GLOVIS) -> ~\IT\ZZ00.SteveNo
  • 현재 배포판으로 Required기능구현 완료.
  • Security(Login) 및 Graphical Presentation 2개 정도 추가 배포 할 수 있슴.

설치 가이드

  1. Project name : asset
  2. Project description
    1. Java(1.8) Spring Boot Project
    2. Mongo DB – Spring Data
    3. Gradle package manager
    4. Thymeleaf template engine
    5. Swagger REST API Doc

  3. How to install (DB and WEB on same server)

    1. Install mongo DB
    2. Create c:\DATA\DT folder for the DB location

    3. Run mongo db

    4. Clone the project using Git

    5. Use Eclipse, IntelliJ or other IDE as you like

    6. How to Run (under “asset folder”)

    7. How to compile (under “asset folder”)

    8. Compiled jar will be located ~/asset/build/libs/XXX.jar

    9. How to Run this Jar file on the production server

  4. Web App is set to use 8090 port : http://localhost:8090

  5. Web App has 1 REST API for logging
    1. Under about tab you can access Swagger-UI page for REST API Doc and test.

  6. How to consume REST API from other application

    1. EX. VELES2PDA (~/veles2pda/menu/top.asp)

    2. Add jquery framework to VELES2PDA (file will be provided)

    3. For cross servers add configuration for jQuery

    4. Using jQuery.ajax, consume REST API of this project

How to set PDA.

  1. Go to control panel, change computer name as you want
  2. Do cold reset and connect local network and check if hostname of the PDA is showing in DHCP, WiFi Controller ….
